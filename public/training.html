Training

A training project bundles your dataset, class list, and trained model in one folder. Pick an existing project from the dropdown, or choose -- new project -- to start a fresh one. Projects are saved under the app data directory in training/<project>/.

Buttons

  • Delete -- remove the current project and all its images.
  • Train -- run training with the selected model, epochs, and image size; live metrics appear in place of the editor.
  • Deploy -- convert the trained model to .tflite (if needed), then for NPU targets compile the quantized model for the chosen accelerator (ST Neural-ART or Arm Ethos-U55) and flash it to the connected board's ROMFS. The CPU target skips compilation and flashes the .tflite as-is.
  • Add Images -- import photos from disk; drag & drop onto the window also works.
  • Auto-Annotate -- launch the annotation helper to label boxes in the dataset.
  • Accept / Reject -- mark the current image's annotation as reviewed; rejected images are excluded from training.

Tips

  • The thumbnail strip shows annotation status: green for accepted, red for rejected, grey for pending.
  • Image size must match the input layer of the deployed model; 192 is a good default for YOLOv8n / YOLO11n.
  • Reading or writing the ROMFS partition can take a long time depending on the partition size and link speed.
  • Press Esc to close.
